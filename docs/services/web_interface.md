# Web-Interface

> **Status:** Planned (v0.8.0) · **Tier:** 1 · **Instances:** Single · **Port:** 8000

Local management console for the Silvasonic recording station. Provides a status dashboard, device management, service configuration, and live audio monitoring — all from a browser, no SSH required.

---

## 1. The Problem / The Gap

*   **No Local UI:** Without a web interface, all configuration (enabling devices, toggling services, viewing status) requires SSH and direct database/Redis access.
*   **No Fleet Visibility:** Monitoring the health of a remote recording station is impossible without a dashboard that aggregates service status, disk usage, and recording activity.

## 2. User Benefit

*   **Status Dashboard:** At-a-glance overview of all services, connected microphones, disk usage, and recent recordings.
*   **Device Management:** Enroll, ignore, or disable microphones. Assign Microphone Profiles.
*   **Service Control:** Enable/disable BirdNET, BatDetect, Weather, Uploader — changes take effect within seconds via State Reconciliation.
*   **Live Listen:** Embedded audio player streaming from Icecast mount points (v0.9.0+).

## 3. Core Responsibilities

### Inputs

*   **Redis (Read+Subscribe):** Reads `silvasonic:status:*` keys for initial state, then subscribes to `silvasonic:status` for live updates.
*   **Database:** Reads `devices`, `microphone_profiles`, `system_services`, `recordings`, `detections` for CRUD views.

### Processing

*   **CRUD API:** FastAPI endpoints for device, profile, service, and recording management.
*   **Real-Time SSE:** Server-Sent Events (`hx-ext="sse"`) proxies Redis Pub/Sub events to the browser for live dashboard updates (CPU, Disk, Recording state).
*   **Server-Side Rendering:** Jinja2 templates + HTMX for dynamic page updates without a SPA build step (ADR-0003).
*   **Authentication UX:** Provides a custom HTML login page to replace native Basic Auth popups.

### Outputs

*   **DB Writes:** Updates `devices.enabled`, `system_services.enabled`, `system_services.status` for desired state changes.
*   **Redis Nudge:** `PUBLISH silvasonic:nudge "reconcile"` after any DB state change to wake the Controller.
*   **HTML Pages:** Rendered pages served to the browser.

## 4. Operational Constraints & Rules

| Aspect           | Value / Rule                                         |
| ---------------- | ---------------------------------------------------- |
| **Immutable**    | No — long-running, but restartable without data loss |
| **DB Access**    | Yes — full CRUD on most tables                       |
| **Concurrency**  | Async event loop (FastAPI + uvicorn)                 |
| **State**        | Stateless — all state in DB and Redis                |
| **Privileges**   | Rootless (ADR-0007)                                  |
| **Resources**    | Low — serves HTML, no heavy computation              |
| **QoS Priority** | `oom_score_adj=0` (default) — Tier 1 infrastructure  |

> [!IMPORTANT]
> The Web-Interface has **no direct control** over containers. It writes desired state to the database and sends a nudge — the Controller reconciles. This is the State Reconciliation Pattern (ADR-0017).

## 5. Configuration & Environment

| Variable / Mount        | Description         | Default / Example |
| ----------------------- | ------------------- | ----------------- |
| `SILVASONIC_WEBUI_PORT` | HTTP server port    | `8000`            |
| `POSTGRES_HOST`, `DB_*` | Database connection | via `.env`        |
| `SILVASONIC_REDIS_HOST` | Redis connection    | `redis`           |

## 6. Technology Stack

*   **Framework:** FastAPI + Uvicorn (ADR-0003)
*   **Templating:** Jinja2
*   **Interactivity:** HTMX + Alpine.js (no SPA, no npm build step)
*   **API Docs:** Swagger / OpenAPI (auto-generated by FastAPI)
*   **Database:** SQLAlchemy 2.0+ async, asyncpg

## 7. UI Views & Frontend Structure (Draft)

1.  **Dashboard (`/`):** Real-time system health (SSE), disk usage, active recorders.
2.  **Devices (`/devices`):** Manage USB microphones, assign profiles, view enrollment state.
3.  **Services (`/services`):** Toggle Tier 2 services (Uploader, BirdNET) and view logs.
4.  **Recordings (`/recordings`):** Browse, filter, playback (WAV), and see ML detection overlays.
5.  **Live Stream (`/live`):** Connect to Icecast mount points for real-time listening.
6.  **Login (`/login`):** Custom authentication gateway layout.

## 8. API Endpoints (Draft)

| Route                       | Method | Purpose                                               |
| --------------------------- | ------ | ----------------------------------------------------- |
| `/api/sse/status`           | GET    | SSE stream of Redis `silvasonic:status:*` changes     |
| `/api/devices/{id}/enroll`  | POST   | Enroll a device and assign a profile (triggers Nudge) |
| `/api/services/{id}/toggle` | POST   | Enable/disable a Tier 2 service (triggers Nudge)      |
| `/api/recordings`           | GET    | Paginated list of recordings (HTMX partials)          |
| `/auth/login`               | POST   | Verifies credentials, issues session cookie/token     |

## 9. Open Questions & Future Ideas

*   Mobile-responsive design priority.
*   Spectrogram viewer: Render spectrograms for recent recordings in-browser.
*   Notification system: Alerts for critical events (disk full, service crash, rare species detection).

## 8. Out of Scope

*   **Does NOT** orchestrate containers (Controller's job).
*   **Does NOT** record or process audio (Recorder / Processor's job).
*   **Does NOT** perform inference (BirdNET / BatDetect's job).
*   **Does NOT** expose a public API — intended for local network / Tailscale access only.
